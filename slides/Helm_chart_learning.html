Cấu trúc Helm Chart, Values & Templates

🚀 Helm là gì?

Helm là package manager cho Kubernetes — giống như apt với Debian hay brew với macOS. Helm giúp bạn đóng gói, version hóa và deploy ứng dụng Kubernetes theo cách nhất quán, tái sử dụng được.

📁 Cấu trúc thư mục Helm Chart

Helm chart là bộ khung gồm metadata, giá trị mặc định và các template render thành manifest Kubernetes.

my-app/
├── Chart.yaml # Metadata: tên, version, appVersion
├── values.yaml # Giá trị mặc định dùng cho templates
├── charts/ # Dependency charts / sub-charts
│ └── redis-17.x.x.tgz
├── templates/ # Manifest Kubernetes dạng Go template
│ ├── deployment.yaml
│ ├── service.yaml
│ ├── ingress.yaml
│ ├── configmap.yaml
│ ├── _helpers.tpl # Named template dùng lại
│ ├── NOTES.txt # Ghi chú hiển thị sau khi install
│ └── hpa.yaml # Horizontal Pod Autoscaler
└── .helmignore # File bỏ qua khi package chart

Ghi nhớ: Chart.yaml là “thẻ nhận diện”, values.yaml là “cấu hình”, và templates/ là “bản thiết kế” render ra Kubernetes manifests.

📄 Chart.yaml — Metadata của Chart

Chart.yaml 
apiVersion: v2                      # Helm 3 dùng v2
name: my-app                        # Tên chart
description: A Helm chart for my app
type: application                   # application | library
version: 0.1.0                      # Chart version (SemVer)
appVersion: "1.0.0"                # App version (hiển thị)
dependencies:
  - name: redis
    version: "17.x.x"
    repository: https://charts.bitnami.com/bitnami
    condition: redis.enabled       # Enable/disable bằng values

⚙️ values.yaml — Cấu hình mặc định

values.yaml 
# Cấu hình replica và image
replicaCount: 2

image:
  repository: nginx
  tag: "1.25.3"
  pullPolicy: IfNotPresent

service:
  type: ClusterIP            # ClusterIP | NodePort | LoadBalancer
  port: 80

ingress:
  enabled: false
  hostname: myapp.example.com
  annotations: {}

resources:
  limits:
    cpu: 500m
    memory: 256Mi
  requests:
    cpu: 100m
    memory: 128Mi

autoscaling:
  enabled: false
  minReplicas: 2
  maxReplicas: 10

redis:
  enabled: true
  auth:
    password: "changeme"    # ⚠️ Không để plaintext trong production!
💡 Thứ tự ưu tiên của Values (thấp → cao) values.yaml mặc định → values.yaml của parent chart → file -f myvals.yaml → tham số --set foo=bar

📋 Templates — Go Template Engine

templates/deployment.yaml 
apiVersion: apps/v1
kind: Deployment
metadata:
  name: {{ include "myapp.fullname" . }}
  labels: {{- include "myapp.labels" . | nindent 4 }}
spec:
  replicas: {{ .Values.replicaCount }}
  selector:
    matchLabels: {{- include "myapp.selectorLabels" . | nindent 6 }}
  template:
    spec:
      containers:
        - name: {{ .Chart.Name }}
          image: "{{ .Values.image.repository }}:{{ .Values.image.tag }}"
          imagePullPolicy: {{ .Values.image.pullPolicy }}
          ports:
            - containerPort: {{ .Values.service.port }}
          resources: {{- toYaml .Values.resources | nindent 12 }}
{{- if .Values.autoscaling.enabled }}
          # HPA sẽ quản lý replicas
{{- end }}
templates/_helpers.tpl 
{{/*
  Named templates — reusable partials
*/}}

{{- define "myapp.fullname" -}}
{{- .Release.Name | trunc 63 | trimSuffix "-" }}
{{- end }}

{{- define "myapp.labels" -}}
helm.sh/chart: {{ .Chart.Name }}-{{ .Chart.Version }}
{{ include "myapp.selectorLabels" . }}
app.kubernetes.io/managed-by: {{ .Release.Service }}
{{- end }}

{{- define "myapp.selectorLabels" -}}
app.kubernetes.io/name: {{ .Chart.Name }}
app.kubernetes.io/instance: {{ .Release.Name }}
{{- end }}

🔍 helm template — Debug trước khi deploy

Terminal 
# Render templates thành YAML thuần (không deploy)
helm template my-release ./my-app

# Với custom values file
helm template my-release ./my-app -f values-prod.yaml

# Override từng giá trị
helm template my-release ./my-app --set image.tag=v2.0.0

# Dry-run với debug (kiểm tra logic phức tạp)
helm install my-release ./my-app --dry-run --debug

# Lint — kiểm tra cú pháp
helm lint ./my-app

⚡ Template Functions thường dùng

String Functions

trunc 63 — cắt chuỗi
trimSuffix "-" — bỏ suffix
upper/lower — đổi hoa/thường
quote — thêm dấu ngoặc
replace "a" "b" — thay thế

Flow Control

if/else/end — điều kiện
range — vòng lặp
with — scope thay đổi
define — named template
include — gọi template

Data Functions

toYaml — convert sang YAML
fromYaml — parse YAML
nindent N — indent + newline
default "val" — giá trị mặc định
required "msg" — bắt buộc

Upgrade, Rollback, Dependency & Secret Management

🔄 Upgrade & Rollback Lifecycle

Revision 1
Initial Install
→
Revision 2
Upgrade A
→
Revision 3
❌ Broken
←
Rollback → R2
Revision 4
Helm Upgrade & Rollback Commands 
# ─── UPGRADE ───────────────────────────────────────────────
# Upgrade lên version mới
helm upgrade my-release ./my-app -f values-prod.yaml

# Upgrade + cài mới nếu chưa tồn tại (idempotent — dùng trong CI/CD)
helm upgrade --install my-release ./my-app -f values.yaml

# --atomic: tự rollback nếu deploy thất bại
helm upgrade --install --atomic --timeout 5m \
  my-release ./my-app -f values-prod.yaml

# Preview thay đổi trước khi apply (dùng helm-diff plugin)
helm diff upgrade my-release ./my-app -f values.yaml

# ─── ROLLBACK ──────────────────────────────────────────────
# Xem lịch sử releases
helm history my-release

# Rollback về revision cụ thể
helm rollback my-release 2

# Rollback về revision ngay trước đó
helm rollback my-release 0

# ─── QUẢN LÝ ───────────────────────────────────────────────
helm list -A                    # Xem tất cả releases
helm status my-release          # Trạng thái release
helm uninstall my-release       # Xóa release
⚠️ Best Practice cho Production Luôn dùng --atomic trong CI/CD. Luôn pin chart version cụ thể (không dùng "latest"). Dùng helm diff để review thay đổi trước khi apply.

📦 Dependency Charts

Chart.yaml — Khai báo dependencies 
dependencies:
  - name: postgresql
    version: "13.x.x"
    repository: https://charts.bitnami.com/bitnami
    condition: postgresql.enabled

  - name: redis
    version: "18.x.x"
    repository: https://charts.bitnami.com/bitnami
    condition: redis.enabled
values.yaml — Config cho subchart 
postgresql:
  enabled: true
  auth:
    username: myuser
    password: mypassword  # → dùng secret!
    database: mydb
  primary:
    persistence:
      size: 10Gi

redis:
  enabled: false
Terminal — Dependency workflow 
# Thêm repo của dependency
helm repo add bitnami \
  https://charts.bitnami.com/bitnami
helm repo update

# Tải dependencies về thư mục charts/
helm dependency update ./my-app

# Kiểm tra dependency tree
helm dependency list ./my-app

# Build (đóng gói)
helm dependency build ./my-app

✅ External Secrets

# Kéo từ AWS/GCP/Vault
apiVersion: external-secrets.io/v1
kind: ExternalSecret
spec:
  secretStoreRef:
    name: aws-secretsmanager
templates/secret.yaml — K8s Secret từ Values 
apiVersion: v1
kind: Secret
metadata:
  name: {{ include "myapp.fullname" . }}-secret
type: Opaque
data:
  # b64enc tự động encode sang base64
  db-password: {{ .Values.database.password | b64enc | quote }}
  api-key: {{ .Values.externalApi.key | b64enc | quote }}

# Cách đúng: dùng existingSecret để reference K8s Secret có sẵn
{{- if not .Values.existingSecret }}
# Tạo secret mới
{{- else }}
# Reference secret đã có: valueFrom.secretKeyRef.name
{{- end }}
💡 Recommended: External Secrets Operator (ESO) ESO là lựa chọn tốt nhất cho production — tích hợp với AWS Secrets Manager, GCP Secret Manager, HashiCorp Vault. Nó có nhiều GitHub stars hơn helm-secrets và Vault Secrets Operator, và hoạt động tốt với ArgoCD.

📋 ArgoCD Application Manifest

argocd/prod/my-app.yaml 
apiVersion: argoproj.io/v1alpha1
kind: Application
metadata:
  name: myapp-production
  namespace: argocd
spec:
  project: default
  sources:
    # Helm chart source
    - repoURL: https://github.com/org/helm-charts
      targetRevision: main
      path: my-app
      helm:
        valueFiles:
          - $values/environments/prod/values.yaml
    # Config repo riêng (values)
    - repoURL: https://github.com/org/gitops-config
      targetRevision: main
      ref: values
  destination:
    server: https://kubernetes.default.svc
    namespace: production
  syncPolicy:
    automated:
      prune: true
      selfHeal: true
    syncOptions:
      - CreateNamespace=true
ApplicationSet — Quản lý 3 môi trường cùng lúc 
apiVersion: argoproj.io/v1alpha1
kind: ApplicationSet
metadata:
  name: myapp-environments
spec:
  generators:
    - list:
        elements:
          - env: dev
            namespace: dev
            revision: HEAD
          - env: staging
            namespace: staging
            revision: main
          - env: prod
            namespace: production
            revision: v1.2.3
  template:
    metadata:
      name: 'myapp-{{env}}'
    spec:
      destination:
        namespace: '{{namespace}}'
      source:
        helm:
          valueFiles:
            - environments/{{env}}/values.yaml
💡 ArgoCD Best Practices Mỗi environment dùng folder riêng trong Git. Pin prod vào tag cụ thể (không dùng HEAD). Không lưu secret plain text — dùng sealed-secrets, external-secrets, hoặc SOPS. Dùng ApplicationSet để tránh lặp lại Application manifest cho nhiều môi trường.

JFrog Artifactory — Helm Chart Repository

🏭 JFrog Artifactory là gì?

JFrog Artifactory là universal artifact repository — lưu trữ Helm charts, Docker images, npm packages trong một nơi. Dùng làm private Helm repository cho team/enterprise.

Làm việc với JFrog Artifactory Helm Repository 
# ─── THÊM REPO JFROG ───────────────────────────────────────
helm repo add my-jfrog \
  https://<server>/artifactory/<repo-name> \
  --username $JFROG_USER \
  --password $JFROG_PASSWORD

helm repo update

# ─── TÌM KIẾM & CÀI ĐẶT ─────────────────────────────────
helm search repo my-jfrog/my-app
helm install my-release my-jfrog/my-app --version 1.2.3

# ─── PUSH CHART LÊN JFROG ────────────────────────────────
# Package chart thành .tgz
helm package ./my-app

# Upload lên Artifactory
curl -u $USER:$PASS -T my-app-0.1.0.tgz \
  https://<server>/artifactory/<repo>/my-app-0.1.0.tgz

# Hoặc dùng JFrog CLI
jf rt upload my-app-0.1.0.tgz <repo-name>/

# ─── SỬ DỤNG TRONG CI/CD ─────────────────────────────────
# GitHub Actions / GitLab CI
helm repo add jfrog-helm \
  "$JFROG_URL/artifactory/$HELM_REPO" \
  --username "$JFROG_USER" \
  --password "$JFROG_TOKEN"

helm upgrade --install --atomic \
  my-app jfrog-helm/my-app \
  --version $CHART_VERSION \
  -f environments/prod/values.yaml

📝 OCI Registry (Helm 3.8+)

# JFrog hỗ trợ OCI format
helm push my-app-0.1.0.tgz \
  oci://<server>/artifactory/<repo>

# Cài đặt từ OCI
helm install my-release \
  oci://<server>/artifactory/<repo>/my-app \
  --version 0.1.0

helm create <name> Tạo chart mới
