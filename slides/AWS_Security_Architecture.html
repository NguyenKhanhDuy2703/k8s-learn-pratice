Pháo đài AWS — nhìn từ trên xuống

Hãy tưởng tượng hạ tầng AWS như một pháo đài nhiều vòng tường. Kẻ tấn công từ Internet phải xuyên qua từng vòng để chạm được dữ liệu ở lõi. Bấm vào từng vòng hoặc từng dòng bên phải để xem service nào trấn giữ ở đó.

01 / 07

Perimeter — Chặn ở cửa ngõ Internet

Đây là tuyến đầu, nơi traffic từ Internet chạm vào hệ thống lần đầu tiên. Mục tiêu: lọc bỏ tấn công DDoS, SQL injection, XSS trước khi chúng chạm tới application.

Internet
Người dùng / kẻ tấn công
→
Shield
Chặn DDoS tự động
→
CloudFront
CDN, edge caching
→
WAF
Lọc Layer 7, OWASP Top 10
→
ALB
Load balancer → app
Lớp này bảo vệ: tính khả dụng của hệ thống (chống DDoS) và tầng ứng dụng web (chống injection, XSS) — trước khi request chạm vào VPC của bạn.
AWS Shield DDoS
Standard: miễn phí, tự động cho mọi tài khoản AWS. Advanced: $3,000/tháng, có team DRT 24/7 hỗ trợ trực tiếp khi bị tấn công lớn.
AWS WAF L7
Soi từng request HTTP: chặn SQL injection trong query string, chặn script độc trong input (XSS), rate-limit để chống credential stuffing.
Amazon CloudFront
CDN phân phối nội dung qua edge location — giảm tải gốc, đồng thời là điểm tích hợp với WAF và Shield ngay tại biên.
AWS Network Firewall
Tường lửa stateful L3/L4/L7: lọc domain, áp luật IPS tương thích Suricata. Triển khai tập trung trong VPC dịch vụ chia sẻ.
02 / 07

Network — Cách ly bằng VPC

Bên trong VPC, traffic được kiểm soát bằng 2 lớp tường lửa khác cấp độ. Nguyên tắc bất biến: database không bao giờ nằm ở public subnet.

Security Group Stateful
Tường lửa ở cấp instance (ENI). Chỉ có luật Allow. Tự động cho phép traffic trả về — đây là tường lửa chính bạn dùng hằng ngày.
Network ACL Stateless
Tường lửa ở cấp subnet. Có cả Allow và Deny, xét theo thứ tự số. Không tự nhớ traffic trả về — phải khai báo cả hai chiều.
VPC Endpoints
Kết nối riêng tới S3, DynamoDB và 100+ service AWS khác mà không cần ra Internet Gateway — traffic ở lại trong mạng AWS.
AWS PrivateLink
Expose service riêng tư giữa các VPC/account khác nhau mà không cần VPC peering hay mở route phức tạp.
Sơ đồ subnet điển hình
PUBLIC SUBNET — 10.0.1.0/24
NAT Gateway · Internet Gateway
PRIVATE SUBNET — 10.0.2.0/24
EC2 / RDS · App servers — không có route ra internet trực tiếp
03 / 07

Identity — Ai được làm gì

Lớp quan trọng nhất vì 39% breach liên quan tới credential bị đánh cắp. Nguyên tắc cốt lõi: Least Privilege — chỉ cấp đúng quyền cần, không hơn.

IAM Roles Ưu tiên
Credential tạm thời, được assume bởi service/app/người dùng liên kết. Ưu tiên hơn IAM User có access key dài hạn.
MFA
Bắt buộc trên root account ngay sau khi tạo. Có thể enforce toàn org bằng SCP — không ai được bỏ qua.
IAM Identity Center
SSO tập trung cho mọi account trong tổ chức. Tích hợp Active Directory, Okta, Azure AD. Cấp credential tạm thời, không có static key.
IAM Access Analyzer
Phát hiện quyền không dùng tới, giúp tinh chỉnh policy về sát với nhu cầu thực tế.
3 cơ chế giới hạn quyền nâng cao
Permission Boundaries
Đặt mức trần quyền tối đa mà một entity (user/role) có thể có, dù policy gắn vào có rộng hơn.
Resource-Based Policies
Gắn trực tiếp lên resource như S3 bucket, KMS key, SQS queue — quyết định ai được phép chạm vào resource đó.
Session Policies
Truyền vào lúc assume role qua STS — giới hạn thêm cho riêng session đó, không ảnh hưởng role gốc.
04 / 07

Compute & Container — Nơi code thực sự chạy

EC2, Lambda, và đặc biệt là EKS (Kubernetes) — mỗi lớp trong stack container cần được khóa riêng.

Control Plane
AWS quản lý, tự động patch. Kết hợp Kubernetes RBAC với IAM. Mã hóa secret trong etcd bằng envelope encryption.
Worker Node
Dùng EKS Managed Node Groups. Bắt buộc IMDSv2 để chặn SSRF. Không SSH — chỉ dùng SSM Session Manager.
IRSA (Pod Identity)
Gán IAM Role riêng cho từng Service Account — pod chỉ có đúng quyền cần, không dùng chung quyền của node.
Network Policies
Calico hoặc VPC CNI network policy: default-deny mọi ingress/egress, chỉ mở traffic pod-to-pod cần thiết.
Falco / GuardDuty for EKS
Phát hiện hành vi container bất thường tại runtime — privilege escalation, API call lạ.
OPA / Kyverno
Policy as code: chặn container chạy privileged ngay tại thời điểm admission, trước khi pod được tạo.
05 / 07

Data — Lõi cần bảo vệ nhất

Dữ liệu được mã hóa ở 3 trạng thái: đang lưu trữ, đang truyền, và đang được xử lý. Đây là lớp trong cùng — nếu mọi lớp ngoài đều bị xuyên qua, mã hóa vẫn là lưới bảo vệ cuối cùng.

At rest
KMS · S3 SSE · EBS · RDS encryption
+
In transit
TLS 1.2/1.3 · ACM · VPN / Direct Connect
+
In use
Nitro Enclaves · Tokenization
Amazon S3 — 6 lớp kiểm soát
Block Public Access
Thiết lập ở cấp account, chặn mọi ACL public — bật vĩnh viễn, không có lý do hợp lệ để tắt.
Bucket Policies
JSON resource-based policy kiểm soát ai truy cập bucket, trong điều kiện nào (IP, MFA, thời gian...).
SSE-S3 / SSE-KMS
Mã hóa server-side dùng key do S3 quản lý hoặc customer-managed key trong KMS để kiểm soát chặt hơn.
S3 Access Logs
Ghi lại mọi request tới bucket — phục vụ audit và điều tra sự cố sau này.
Object Lock (WORM)
Chặn xóa object trong thời gian retention — lớp phòng thủ quan trọng chống ransomware.
Amazon Macie
Dùng ML để tự động phát hiện PII và dữ liệu nhạy cảm nằm rải rác trong các bucket S3.
Quản lý khóa — KMS & Secrets Manager
AWS KMS
HSM đạt chuẩn FIPS 140-2. 3 loại key: AWS Managed (free, tự xoay), Customer Managed (toàn quyền kiểm soát), AWS Owned (chia sẻ, không thấy được).
AWS Secrets Manager
Lưu credential database, API key, token. Tự động xoay bằng Lambda — loại bỏ hoàn toàn mật khẩu fix cứng trong code.
06 / 07

Watch — Nhìn thấy mọi thứ xảy ra

Lớp này bao trùm toàn bộ pháo đài — không chặn tấn công, nhưng đảm bảo bạn biết khi có chuyện xảy ra và chứng minh được điều đó với auditor.

CloudTrail
Ghi mọi API call — ai làm gì, lúc nào, từ đâu. Câu hỏi kinh điển "ai đã xóa bucket này?" luôn được trả lời từ đây.
CloudWatch
Thu thập metric, log, event theo thời gian thực. Alarm khi CPU spike bất thường hay tỷ lệ lỗi API tăng vọt.
AWS Config
Theo dõi thay đổi cấu hình theo thời gian. 140+ rule có sẵn, ví dụ "S3 bucket không được public".
GuardDuty ML
Phân tích CloudTrail, VPC Flow Log, DNS log bằng machine learning để phát hiện credential bị đánh cắp, đào coin trái phép, port scan.
Security Hub
Single pane of glass — tổng hợp finding từ GuardDuty, Inspector, Macie và đối chiếu với CIS, PCI DSS, NIST 800-53.
Amazon Detective
Điều tra dạng đồ thị — trả lời câu hỏi "việc này xảy ra như thế nào?", giảm đáng kể thời gian điều tra (MTTR).
Amazon Inspector
Quét lỗ hổng liên tục cho EC2 (qua SSM agent), Lambda (package dependency), và image trong ECR.
Audit Manager
Tự động thu thập bằng chứng đối chiếu CIS, NIST, PCI DSS, SOC2, ISO 27001, HIPAA — giảm 90% thời gian chuẩn bị audit.
07 / 07

DevSecOps — Bảo mật trước khi lên Production

Shift Left: sửa lỗi ở bước code rẻ hơn rất nhiều so với sửa ở production. Mỗi giai đoạn pipeline có một cổng kiểm tra riêng.

Code
  • SAST — quét lỗ hổng mã nguồn tĩnh (SQLi, …)
  • Secret scanning — phát hiện mật khẩu fix cứng
  • Peer review
Build
  • Dependency scan — đánh giá an toàn thư viện mã nguồn mở
  • License check — kiểm tra license được phép dùng
  • SBOM generation — bản kê toàn bộ thành phần phần mềm
Test
  • DAST — kiểm thử bảo mật bằng tấn công giả
  • Pen testing — tìm lỗ hổng luồng nghiệp vụ
  • IaC scan — kiểm tra mã hạ tầng (Terraform, CFN)
Package
  • Container scan — tìm lỗ hổng trong image
  • Image sign — đóng dấu, đảm bảo image không bị sửa
  • Registry push — đẩy lên ECR (immutable tag)
Deploy
  • Policy gate — chặn nếu vi phạm chính sách
  • Config check — kiểm tra port/cấu hình mở nhầm
  • Drift detect — phát hiện lệch cấu hình giữa môi trường

Công cụ theo từng giai đoạn

Checkov / tfsec / cfn-nag
Quét Terraform, CloudFormation, Kubernetes manifest tìm lỗi cấu hình bảo mật.
Gitleaks / truffleHog
Quét lịch sử git tìm secret đã commit nhầm trong quá khứ.
Cosign / Sigstore
Ký số và xác minh container image trước khi deploy lên Kubernetes.
OWASP ZAP / Burp Suite
Kiểm thử bảo mật ứng dụng web đang chạy thực tế (DAST).

AWS Organizations + Service Control Policies (SCP) thiết lập giới hạn quyền tối đa áp dụng xuyên suốt mọi tầng kể trên — không account nào, kể cả root, được vượt qua.

SCP — Service Control Policy chỉ giới hạn, không bao giờ cấp quyền

  1. Chặn rời khỏi AWS Organization
  2. Bắt buộc MFA để xóa S3 bucket
  3. Chặn dùng service ngoài region được duyệt
  4. Chặn tắt CloudTrail hoặc GuardDuty
  5. Bắt buộc IMDSv2 trên mọi EC2
  6. Chặn tạo IAM user có console access

2

3
Investigation

4
Recovery

Lessons Learned
